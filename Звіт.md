# Звіт до індивідуального завдання з теми «Структура + Файл»
## 1. Умова задачі
Зробити програму, яка моделює редактор для створення та редагування табло автовокзалу. На Табло повинен бути заголовок, та список рейсів з відповідною інформацією (номер зупинки, час, номер автобуса і т.д.).

### Функціональність
**Загальне призначення:** Програма імітує редактор для створення або редагування вигляду інформаційного електронного табло, відображаючи структурований розклад відправлення автобусів, заголовок та час (задається вручну).

**1. Редагування інформації**
Основне завдання програми — додавання, редагування та видалення наступних об’єктів з екрану:
- довільного тексту (заголовку, назв колонок, ...)
- ліній для розмітки (горизонтальних або вертикальних)
- слотів розкладу (час відправлення, номери автобуса та зупинки, статус рейсу)

**2. Зчитування та зберігання Табло**
Програма може зчитувати вже існуюче табло з `.txt` файлу. Також є можливість створити нове Табло (новий файл) та зберегти зміни у файл. Існує файл `config.txt`, в якому можна задати параметри програми - максимальний розмір текстових полів, розміри та фоновий символ екрану, а також відступи між полями слоту.

**3. Виведення Табло в термінал**
Для перевірки змін програма виводить табло в термінал. Під час виводу програма перевіряє об’єкти на коректність (значення полів структури `Sprite`) та на перетини між собою та з границями екрану (колізії).

**Вибір Структур**
Для зручного створення та змінення вигляду інформації на екрані будь-який об’єкт (заголовок, лінія або пункт розкладу) представлено структурою `Sprite`. У кожного спрайту є назва, координати (числові поля `x`, `y`), тип спрайту (поле `enum TypeOfSprite`) та варіативна частина для зберігання деталей відповідного типу (поле `union DetailsType`).

## 2. Опис специфікації всіх функцій

### Головна логіка та меню (`departure_board_editor.h`)
* `void start();` — Ініціалізація програми, завантаження конфігурації з `config.txt` та запуск головного циклу меню (state machine).

### Робота з одним об'єктом (Спрайтом) (`one_sprite.h`)
* `void set_actual_departure_time(Sprite *sprite, TimeType depTime);` — Функція, яка описує поведінку об'єкту. Встановлює реальний час відправлення відповідному рейсу (слоту).
* `int measure_sprite_length(Sprite *sprite);` — Підраховує розмір спрайту (кількість символів на екрані).
* `bool validate_sprite(const Sprite *sprite);` — Перевіряє коректність полів спрайту.
* `bool validate_details(DetailsType *details, TypeOfSprite type);` — Перевіряє коректність варіативної частини (union).
* `char *slot_to_string(const ScheduleSlot *slot, char bg_char);` — Формує рядкове представлення слоту розкладу.

### Робота з масивом об'єктів та пошук (`many_sprites.h`)
* `bool add_sprite_to_sprites(const Sprite *sprite);` — Додає новий спрайт до масиву.
* `bool edit_sprite_in_sprites();` — Редагування обраного спрайту.
* `bool delete_sprite_from_sprites(int sprite_index);` — Видалення спрайту.
* `int sprites_len(Sprite **sprites);` — Повертає кількість спрайтів.
* `Sprite *find_sprites_by_model(Sprite **sprites, Sprite model);` — **Пошук за вказаним значенням поля `enum` та `union`**. Знаходить спрайти, які відповідають полям макетного спрайту (пошук за типом та специфічними деталями).
* `Sprite *find_sprites_by_coords_diapason(Sprite *sprites, int x1, int y1, int x2, int y2);` — **Пошук за числовим полем**. Знаходить спрайти, координати яких потрапляють в заданий діапазон (прямокутник).

### Введення та виведення, робота з файлами (`input_output.h`)
* `void input_sprite_from_stdin(Sprite *sprite);` — Введення полів основної структури з клавіатури, що викликає функції для введення `enum` та `union`.
* `Sprite **input_sprites_from_file(const char *filename, bool *is_file_successfully_opened);` — Зчитування інформації з файлу.
* `void output_sprites_to_stdout(Sprite **sprites);` — Виведення інформації про масив спрайтів у консоль.
* `void output_sprites(Sprite **sprites, FILE *dest);` — Запис масиву спрайтів у файл (збереження табло).
* `void free_sprites_array(Sprite **sprites);` — Очищення пам'яті.

### Відображення на екрані та колізії (`display.h`)
* `bool display_sprites(bool test_mode);` — Виводить всі спрайти на екран. У `test_mode` виводяться осі координат.
* `ScreenType *check_sprites(ScreenType screen);` — Перевіряє всі спрайти на перетини між собою (колізії) та вихід за межі екрану.
* `void fill_screen(ScreenType *screen);` — Заповнює буфер екрану фоновим символом.

### Допоміжні функції (`help_functions.h`)
* `void remove_newline(char *str);`, `void clear_stdin();`, `bool find_field_in_file(FILE *file, const char *field);` — Функції для безпечного читання вводу та парсингу конфігураційних файлів.

## 3. Текст програми з коментарями

### `globals_and_structures.h` (Основні структури)
```c
#ifndef STRUCT_FILE_DB_STRUCTURES_H
#define STRUCT_FILE_DB_STRUCTURES_H

// Структура, яка відповідає тексту
struct Text {
    char *content;
};

// Перерахування для напрямку лінії
typedef enum {
    RIGHT = 1,
    DOWN
} DirectionEnum;

// Структура, яка відповідає лінії
struct Line {
    char character;
    int length;
    DirectionEnum direction;
};

// Статус рейсу
typedef enum {
    WAITING = 1,
    IN_PROGRESS,
    CANCELLED
} SlotStatus;

// Час
typedef struct {
    int hours;
    int minutes;
} TimeType;

// Структура, яка відповідає одному слоту в розкладі
typedef struct Slot {
    int station_number;
    TimeType scheduled_departure; // запланований час відправлення
    TimeType estimated_departure; // передбачуваний час відправлення
    int trip_number;
    SlotStatus status;
} ScheduleSlot;

// Тип спрайту (обов'язкове поле типу enum)
typedef enum {
    TEXT = 1,
    LINE,
    SLOT
} TypeOfSprite;

// Об'єднання всіх типів спрайту (обов'язкове поле типу union)
typedef union {
    struct Text text;
    struct Line line;
    struct Slot slot;
} DetailsType;

// Головна структура Спрайту (об'єкт)
typedef struct {
    char *name;
    int x;          // числове поле
    int y;          // числове поле
    TypeOfSprite type;      // поле типу enum
    DetailsType details;    // поле типу union (вкладена структура)
} Sprite;

// ... інші допоміжні структури (ScreenType, MenuEnum, StateType) ...

#endif //STRUCT_FILE_DB_STRUCTURES_H
```

### `main.c`
```c
#include <stdlib.h>
#include "departure_board_editor.h"

int main(void) {
    // Запуск головної логіки редактора табло
    start();
    return 0;
}
```

*Повний текст програми знаходиться у відповідних `.c` та `.h` файлах проекту, що поділені на модулі `functions/src` та `functions/include`.*
